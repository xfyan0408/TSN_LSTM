是的，代码实现上可以按这个顺序做：

```text
1. 定义模型结构
2. 准备数据集
3. 训练模型
4. 保存训练好的模型
5. 推理时加载模型
6. 输入最新历史窗口，输出未来 H 步需求
```

你的文档里的结构就是：**ChannelMix → TCN → LSTM → 三个输出头**，输入是过去 $T$ 步的带宽/CPU/内存需求，输出是未来 $H$ 步三维资源需求。

# 2. `dataset.py` 写什么

这个文件负责把原始时间序列变成训练样本。

假设原始数据是：

```text
time, bandwidth, cpu, memory
1,    ...
2,    ...
3,    ...
```

模型需要的样本是：

```text
输入 X: 过去 T 步，形状 [T, 3]
标签 Y: 未来 H 步，形状 [H, 3]
```

也就是滑动窗口：

```text
X = data[i : i+T]
Y = data[i+T : i+T+H]
```

`dataset.py` 主要写：

```python
# dataset.py

import torch
from torch.utils.data import Dataset
import pandas as pd
import numpy as np


class ResourceDataset(Dataset):
    def __init__(self, csv_path, window_size, pred_horizon, mean=None, std=None):
        df = pd.read_csv(csv_path)

        # 只取三列资源需求
        values = df[["bandwidth", "cpu", "memory"]].values.astype(np.float32)

        # 训练集计算 mean/std；验证/测试/推理复用训练集 mean/std
        if mean is None:
            mean = values.mean(axis=0)
        if std is None:
            std = values.std(axis=0) + 1e-6

        self.mean = mean
        self.std = std

        values = (values - mean) / std

        self.values = values
        self.window_size = window_size
        self.pred_horizon = pred_horizon

    def __len__(self):
        return len(self.values) - self.window_size - self.pred_horizon + 1

    def __getitem__(self, idx):
        x = self.values[idx : idx + self.window_size]
        y = self.values[idx + self.window_size : idx + self.window_size + self.pred_horizon]

        return torch.tensor(x), torch.tensor(y)
```

注意：**mean/std 必须保存下来**。推理时要用训练时的 mean/std，否则输入尺度不一致。

# 3. `model.py` 写什么

这里定义模型。

输入：

```text
x: [batch, T, 3]
```

经过 ChannelMix：

```text
[batch, T, 3] → [batch, T, D]
```

TCN 需要 Conv1d 格式：

```text
[batch, T, D] → [batch, D, T]
```

TCN 输出后再转回 LSTM：

```text
[batch, D, T] → [batch, T, D]
```

最后 LSTM 输出窗口表示，再接三个 Head。

```python
# model.py

import torch
import torch.nn as nn


class TCNBlock(nn.Module):
    def __init__(self, channels, kernel_size=3, dilation=1):
        super().__init__()
        padding = (kernel_size - 1) * dilation

        self.conv = nn.Conv1d(
            in_channels=channels,
            out_channels=channels,
            kernel_size=kernel_size,
            dilation=dilation,
            padding=padding
        )

        self.relu = nn.ReLU()

    def forward(self, x):
        # x: [B, C, T]
        out = self.conv(x)

        # 去掉右侧多出来的 padding，保证因果卷积只看历史
        out = out[:, :, :x.size(-1)]

        return self.relu(out + x)


class ResourcePredictor(nn.Module):
    def __init__(self, input_dim=3, hidden_dim=64, tcn_layers=3, lstm_hidden=64, pred_horizon=10):
        super().__init__()

        self.channel_mix = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU()
        )

        self.tcn = nn.Sequential(
            *[
                TCNBlock(
                    channels=hidden_dim,
                    kernel_size=3,
                    dilation=2 ** i
                )
                for i in range(tcn_layers)
            ]
        )

        self.lstm = nn.LSTM(
            input_size=hidden_dim,
            hidden_size=lstm_hidden,
            batch_first=True
        )

        self.head_b = nn.Linear(lstm_hidden, pred_horizon)
        self.head_c = nn.Linear(lstm_hidden, pred_horizon)
        self.head_m = nn.Linear(lstm_hidden, pred_horizon)

    def forward(self, x):
        # x: [B, T, 3]

        z = self.channel_mix(x)
        # z: [B, T, D]

        z = z.transpose(1, 2)
        # z: [B, D, T]

        z = self.tcn(z)
        # z: [B, D, T]

        z = z.transpose(1, 2)
        # z: [B, T, D]

        _, (h_n, _) = self.lstm(z)

        s = h_n[-1]
        # s: [B, lstm_hidden]

        pred_b = self.head_b(s)
        pred_c = self.head_c(s)
        pred_m = self.head_m(s)

        # [B, H, 3]
        out = torch.stack([pred_b, pred_c, pred_m], dim=-1)

        return out
```

这个模型已经符合你的设计路线。

# 4. `train.py` 写什么

训练入口主要做：

```text
1. 读取数据
2. 构造 Dataset / DataLoader
3. 创建模型
4. 定义 loss 和 optimizer
5. 训练循环
6. 保存 best_model.pt
```

```python
# train.py

import torch
import torch.nn as nn
from torch.utils.data import DataLoader, random_split
import os

from config import *
from dataset import ResourceDataset
from model import ResourcePredictor


def train():
    os.makedirs("checkpoints", exist_ok=True)

    dataset = ResourceDataset(
        csv_path=DATA_PATH,
        window_size=WINDOW_SIZE,
        pred_horizon=PRED_HORIZON
    )

    train_size = int(len(dataset) * 0.8)
    val_size = len(dataset) - train_size

    train_set, val_set = random_split(dataset, [train_size, val_size])

    train_loader = DataLoader(train_set, batch_size=BATCH_SIZE, shuffle=True)
    val_loader = DataLoader(val_set, batch_size=BATCH_SIZE, shuffle=False)

    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

    model = ResourcePredictor(
        input_dim=INPUT_DIM,
        hidden_dim=HIDDEN_DIM,
        tcn_layers=3,
        lstm_hidden=LSTM_HIDDEN,
        pred_horizon=PRED_HORIZON
    ).to(device)

    criterion = nn.MSELoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=LR)

    best_val_loss = float("inf")

    for epoch in range(EPOCHS):
        model.train()
        train_loss = 0.0

        for x, y in train_loader:
            x = x.to(device)
            y = y.to(device)

            pred = model(x)
            loss = criterion(pred, y)

            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            train_loss += loss.item()

        model.eval()
        val_loss = 0.0

        with torch.no_grad():
            for x, y in val_loader:
                x = x.to(device)
                y = y.to(device)

                pred = model(x)
                loss = criterion(pred, y)

                val_loss += loss.item()

        train_loss /= len(train_loader)
        val_loss /= len(val_loader)

        print(f"Epoch {epoch+1}/{EPOCHS}, train={train_loss:.6f}, val={val_loss:.6f}")

        if val_loss < best_val_loss:
            best_val_loss = val_loss

            torch.save({
                "model_state_dict": model.state_dict(),
                "mean": dataset.mean,
                "std": dataset.std,
                "config": {
                    "input_dim": INPUT_DIM,
                    "hidden_dim": HIDDEN_DIM,
                    "lstm_hidden": LSTM_HIDDEN,
                    "pred_horizon": PRED_HORIZON,
                    "window_size": WINDOW_SIZE
                }
            }, CKPT_PATH)

            print(f"Saved best model to {CKPT_PATH}")


if __name__ == "__main__":
    train()
```

严格来说，时间序列更推荐按时间顺序切训练/验证，不要随机切。你刚开始写代码可以先这样跑通，后面再改成前 80% 训练、后 20% 验证。

# 5. `infer.py` 写什么

推理入口主要做：

```text
1. 加载 best_model.pt
2. 创建同样结构的模型
3. 加载权重
4. 读取最近 T 步资源需求
5. 用训练时 mean/std 归一化
6. model.eval() 推理
7. 反归一化输出
```

```python
# infer.py

import torch
import pandas as pd
import numpy as np

from config import CKPT_PATH
from model import ResourcePredictor


def load_model():
    ckpt = torch.load(CKPT_PATH, map_location="cpu")
    cfg = ckpt["config"]

    model = ResourcePredictor(
        input_dim=cfg["input_dim"],
        hidden_dim=cfg["hidden_dim"],
        lstm_hidden=cfg["lstm_hidden"],
        pred_horizon=cfg["pred_horizon"]
    )

    model.load_state_dict(ckpt["model_state_dict"])
    model.eval()

    return model, ckpt


def predict_recent(csv_path):
    model, ckpt = load_model()

    mean = ckpt["mean"]
    std = ckpt["std"]
    window_size = ckpt["config"]["window_size"]

    df = pd.read_csv(csv_path)
    values = df[["bandwidth", "cpu", "memory"]].values.astype(np.float32)

    recent = values[-window_size:]

    recent_norm = (recent - mean) / std

    x = torch.tensor(recent_norm).unsqueeze(0)
    # x: [1, T, 3]

    with torch.no_grad():
        pred_norm = model(x).squeeze(0).numpy()
        # [H, 3]

    pred = pred_norm * std + mean

    return pred


if __name__ == "__main__":
    pred = predict_recent("data/resource.csv")

    print("未来资源需求预测:")
    print("columns: bandwidth, cpu, memory")
    print(pred)
```

# 8. 最小主线

你现在可以记住这条主线：

```text
训练阶段：
resource.csv
    → Dataset 切滑动窗口
    → model(X)
    → pred: [batch, H, 3]
    → 和真实未来 Y 计算 loss
    → 反向传播
    → 保存 best_model.pt

推理阶段：
最近 T 步资源需求
    → 使用训练时 mean/std 归一化
    → 加载 best_model.pt
    → model.eval()
    → 输出未来 H 步预测
    → 反归一化
    → 交给资源可行性检测
```

所以你的理解基本正确：

> 先定义模型，然后训练时调用模型，训练完成后保存模型；推理时加载训练好的模型，再输入最新历史窗口得到未来预测结果。

关键不要漏掉两点：

1. **训练时保存的不只是模型权重，还要保存归一化用的 mean/std。**
2. **推理时模型结构必须和训练时完全一致，否则权重加载不了。**
